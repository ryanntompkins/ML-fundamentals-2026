### Logistic Regression uses regularization and need comparable feature scales. Since the numerical values in this data set exist on different scales it is important to basically translate them in a way that the model will better understand. Without scaling, features with larger numerical ranges would dominate the gradient updates and exert disproportionate influence on the model.

## 6.4 Fitted on Training Set

### The scaler must be fitted on the training set only because it estimates parameters.If fitted on the entire dataset, information from validation and test sets would influence the scaling parameters, resulting in data leakage.

## 6.5 Affects of Feature Scaling

### Gradient-Based Optimization: Scaling ensures stable and efficient convergence. Without scaling gradients with larger scales would dominate.

### Coefficient Comparability: Without scaling, coefficients are not comparable. This allows the coefficients to truly represent a stronger predictive influence.

### Regularization: Regularization penalizes large coefficients. If features are on different scales, regularization unfairly penalizes some features more than others.

This imbalance is substantial and may bias a classifier toward predicting the majority class. A classifier predicting only “no” would already achieve high accuracy (~89%), making accuracy alone an unreliable evaluation metric.

## 8.2 Reassembling Strategy

from imblearn.over_sampling import SMOTE

smote = SMOTE(random_state=42)

X_train_resampled, y_train_resampled = smote.fit_resample(
    X_train,
    y_train_binary
)

print("\nBefore SMOTE:")

print(y_train_resampled.value_counts(normalize=True))
## 8.3 Justification
### There is a deep imbalance between "yes" and "no", this imbalance may create bias in the Logistic Regression Model. SMOTE was selected as the resampling method because it generates examples rather than duplicating the minority, reducing the risk of overfitting.
## 8.4 Implications of Model Evaluation
### If resampling was applied before splitting, data leakage would be present. By applying SMOTE only to the training set, the validation and test sets remain representative of real-world data, preserving the integrity of model evaluation.
## 8.5 Effect of evaluation metrics
### 1. Accuracy: Due to imbalance accuracy can be misleading, this is due to if they only predict "no" it could still be ~89% accurate.
### 2. Precision: Under imbalance, precision may decrease if the model predicts many false positives.
### 3. Recall: Missing potential subscribers due to false positives would destroy the marketing campaugn.

# 10. Order
### When deciding the order it is important to weigh data leakage and which step is needed before the others. Through this methodology, I determined the order of tasks.
### It was clear to me that data loading and exploration came first, it is essential to understand the data you are working with before doing anything else, and you literally cannot without loading in the data. This made it easy to determine that loading and exploring the data was the first step.
### Next comes identifying the prediction target. Once you understand the data through exploration it becomes clear that "yes" is the prediction target, knowing that this campaign wants to hear that "yes" made it easy to understand that was the target. This also becomes later steps because you must know your goal before moving forward with anything else.
### Once you understand the data and goal it's time to begin work: beginning with data splitting to ensure that there is no data leakage before later steps. This step must come before others because keeping all the data in one data frame can lead to leakage and lead the model astray.
### After the splitting you must now start focusing on the training set, beginning with handling missing values. This step can't come before the previous because the data must be split to not cause leakage, and can't come after later steps because you have to handle any missing values before really getting to work on the data and training the model.
### After handling missing values it's time to work on those categorical variables. This has to come after missing values because you have to start encoding them in a way so it will work for training, which will be corrupted if done before handling missing values. It also must come before scaling, selection, etc. because the categorical variables must be put in a way that the program can understand using one-hot for nominal and ordinal encoding for ordinal variables.
### Once categorical and missing values are handled it's time to move in on tuning the data for the Logistic regression Model (LRM), scaling features. This must be done because LRMs need comparable magnitudes and uses regularization. This must come before feature selection because it needs to be comparable to other features so you know which ones to select.
### Obviously next is feature selection, here you can tune down the variables to be used in the model by looking at their coorelations and variance. Here is where you really have to focus on not allowing data leakage. This is important before moving on to imbalance and training because it ensures that all the variables are handled accordingly before training the model (I definitely messed up this step).
### The final step before training the model is addressing class imbalance. Again this is done on only the training set using SMOTE to ensure that you aren't oversampling and causing data leakage before you train the model.
### After everything else is handled you can finally train your model, similar to the data exploration it is clear that this is the last step. This is where you get the final LRM and can see how well your model is performing by comparing it to your test set.

# 11. Final Discussion
### The goal of the Linear Regression Model is to predict whether a client subscribes to a term deposit. This pipeline is meant to make this decision through statistical theory. The prediction target in this case is "y", which means the client subscribes. To ensure its accuracy variables, such as duration in this case, must be excluded that could be predictive. Duration is predictive because it measures the length of the actual call, a longer call signifies that the client does subscribe so it must be excluded from the model. A model that uses variables such as this, while appearing to be highly predictive, are not a valid model. The dataset was split into training, validation, and test sets using stratified sampling. A stratified set is necessary because the result "y" may be underrepresented if other sampling techniques are used. This split was done before any transformations, allowing the validation and test sets to not be influenced from these transformations and allows the test set to not be influenced by the others. This prevents data leakage from the other sets, which would produce many false positives and make the model not independent from them. The dataset contained mostly implicit missing values in the form of "unknown", missing data can have meaning so these occurrences were not dropped. To handle categorical variables you must first determine which are ordinal and which are nominal. There are two different coding techniques for these. For the ordinal variable (education) a map was made to give meaning to the different levels of education. For the nominal variables one-hot encoding was used to give them meaning without creating a hierarchy within them. This distinction is critical for a LRM because it creates the necessary hierarchy for ordinal variables and gives dimensionality for the nominal values, basically translating them in a way the LRM can understand. Feature scaling through standardization was then used, this regularizes the values and allows their coefficients to be comparable which allows the LRM to compare the variables to one another when predicting. If features are not scaled, variables with larger numeric ranges receive disproportionate influence. Feature selection was done with a mix of conceptual and statistical reasoning. To do this you look at variance and collinearity to determine which variables stand on their own when determining a "y" or "n". Variables that had low variance were dropped, this was done to improve the accuracy of the LRM as they signify not contributing to the end result. After these were dropped "duration" was also dropped due to the reasons previously states, it contains information that happens after the call started. The training set shows significant class imbalance with far more "n" than "y", making it necessary to address the class imbalance. This resulted in SMOTE being applied to the training set. SMOTE synthesizes minority examples by interpolating in feature space, reducing overfitting risk compared to simple duplication. This was done after splitting because also performing it on the other sets would inflate performance and cause data leakage. After this the LRM was trained with the training dataset. The accuracy comparison, found by calculation of true positives, true negatives, false positives, and false negatives, returned ~84% which is low for the baseline. However, since there were large class imbalances other metrics are also needed. The precision returns 0.34, calculated True Positive divided by the sum of True Positive and False Positive, meaning it was correct 34% of the time when predicting if a client would subscribe. This shows the presence of false positives. The recall returns 0.485, calculated by True Positive divided by the sum of True Positive and False Negative, meaning identifies 48.5% of new subscribers successfully. The Zero Rules comes in at .89, which when you subtract it from the accuracy you return .05, signifying that the model is actually around 5% lower in accuracy. These results signify that the model correctly predicts anout half of the subscribers. These results lead to the question of SMOTE was justified, the precision is not where I want it to be so were the transformations the correct call? The SMOTE created more of the minority class, allowing the decision to be more balanced. Since it was done on only the training set, I believe that the SMOTE was justified in this case. Although the final accuracy is lower than the Zero Rule baseline, the model provides substantially greater practical value by identifying nearly half of true subscribers.
